TECHNICAL NOTES · 大圖校稿階段
① Header 選單前台未顯示
後台查詢店家帳號 → 進入「其他頁面」選項
新增首頁選單要開啟的欄位 → 前台即會顯示
若後台相關欄位都已開啟但前台仍無顯示，需檢查「其他頁面」內是否有新增對應欄位
② Footer 欄位修改 / 排列順序
Footer 內容由系統自動抓取後台資料，無須手動新增
IG、FB 等社群連結不放在 Footer，統一處理在浮動按鈕
LINE 若有名稱可放入 Footer（放之前先確認客戶 LINE 可正常查詢）
需新增額外欄位 → 從後台的「第二地址」欄位新增內容；新增後前台項目標題會顯示為「地址」，需透過 CSS 修改名稱顯示
CSS 改完名稱後，記得貼到 custom.css 內並上傳到後台
調整排列順序：進 HTML 找到對應項目 → 在 CSS 內對該 li 設定 display: flex → 找到對應項目（如 p、tel）→ 設定 order: 1（數字越小越前面）→ 已設定的項目會排在未設定的項目前面
③ 修改浮動按鈕圖示與連結
從後台的「浮動按鈕 Text」欄位修改
所有圖片（包含 Banner、浮動按鈕圖示）一律上傳至 fz 站台
程式碼從 fz 站台抓取檔案，程式碼內的檔名必須與上傳的檔名完全一致才能正確抓到
若刪掉程式碼但前台仍有顯示 → 按 F12 開啟開發者工具，確認是哪條程式碼仍在作用，再刪除
④ 主機連線方式 / 新增站台
主機選擇 03 → 進入「全部設計師的站台」→ 以 帳號 搜尋找到自己的專案站台
新開專案需自行新增站台：對「全站站台」按右鍵 → 新增 → 輸入帳號 → 完成建立
若在站台列表找不到專案，確認是否選到正確主機（03）以及帳號是否輸入正確
⑤ 內頁 / 首頁上傳方式（含常見問題處理）
進入該案件後台的「內頁區塊」
將新手包內的程式碼（檔案名稱：平面校稿）貼入編輯區塊
將圖片名稱換成該欄位對應的正確圖片名稱
▸ 狀況一：Banner 與首頁內容之間多出一塊白色區塊
按 F12 開啟開發者工具 → 選取該白色區塊
在 CSS 加上 display: none 取消顯示即可
此為系統既有問題，原因不明，以 CSS 隱藏處理即可
完成後記得將 custom.css 的內容複製到後台的 CSS 欄位內，確保樣式正確套用
▸ 狀況二：功能頁需穿插在編輯頁之間（較少見）
將程式碼區塊拆分為「功能頁前半段」與「功能頁後半段」分別儲存
在該案件編輯器內上傳其中一個檔案
在後台開啟「自定義區塊」，將另一個檔案放上去
後台可移動區塊的顯示位置來調整順序
若功能頁需顯示在首頁上 → 需開啟後台的「在首頁顯示」選項（未開啟則客戶修改後台資料後，內容不會同步更新到首頁）
若版本不支援此做法（版本資訊僅供稿師/切版師可見）→ 先將功能頁放在最下面，正式處理等到切版時請切版人員協助
▸ 補充：內頁可直接修改 CSS
⑥ 內頁 Banner 製作與修改
內頁 Banner 換圖一律用 custom.css 修改，不動模組本身
Menu 樣式為模組自帶，若需調整也要寫在 custom.css 內覆蓋
Banner 製作時先裁好想要的尺寸與顯示樣式
▸ 單一內頁 Banner 修改
用該頁面模組的專屬名稱（class）單獨撰寫，避免影響其他頁面
範例：大自然蒸餾水產品頁 → .banner.banF { ... }
背景圖片連結格式：https://主機網址/帳號/圖檔名稱.JPG（可直接複製現有連結修改）
▸ 跨內頁共用設定（如圖片大小）
直接寫在 .banner 父層內，可統一覆蓋所有子層的設定
▸ 調整 Banner 大小
方法一：直接修改圖片尺寸
方法二：調整文字的 padding（適合需要自由排版：置左／置中／置右，用方框內的 padding 推移）
▸ 常用 CSS 範例
background-image: url(https://pic03.eapple.com.tw/048233289/banF.jpg);
background-repeat: no-repeat;
background-position: center center;
所有修改都要寫在 custom.css 內，否則重整後預設模組樣式會覆蓋設定
撰寫過程中若出現亂碼，貼回 VSCode 修改後再貼回即可
所有功能頁的 CSS 一律寫在 Header／Footer 區域
